Publishing Flow

之後的維護流程

加一篇報告時只需要補內容檔和資料檔，首頁與部署流程會一起跟上。

01

新增報告 HTML

把新內容放到對應目錄，例如 `f1/2026/` 或 `baseball/2026/`。

02

更新資料檔

在 `data/reports.json` 補上標題、日期、摘要與連結，首頁就知道要顯示什麼。

03

Push 後自動部署

GitHub Actions 會自動 build `index.html` 並發布到 GitHub Pages。
